Newton Inventory Service API¶

Quick Reference¶

Production url - http://inventory.encore.rackspace.com/1.0

For staging or testing environments, prepend ‘staging.’ or ‘qe.’ to url, respectively.

Authentication¶

How to authenticate with Newton Inventory Service

Products¶

Registry of available families of which a Device is a member.

URL HTTP Verb Functionality
/products GET List all registered Product
/products POST Register a Product
/products/<product_id> GET Retrieve a Product’s details
/products/<product_id> PUT Update a Product
/products/<product_id> DELETE Unregister a Product

Devices¶

Dedicated hardware instances which are tracked in Inventory.

URL HTTP Verb Functionality
/devices GET List all Devices in Inventory
/devices POST Add a Device to Inventory
/devices/<device_id> GET Retrieve a Device’s details from Inventory
/devices/<device_id> DELETE Remove a Device from Inventory

Inventory¶

The collection of all Devices currently tracked in Inventory.

URL HTTP Verb Functionality
/inventory GET Retrieve summarized Inventory counts

Deploy¶

Allocate devices for customers and manage those devices in Inventory.

URL HTTP Verb Functionality
/deploy POST Request Devices and flag them as allocated

Admin¶

Allocate devices for customers and manage those devices in Inventory.

URL HTTP Verb Functionality
/admin GET View current configuration settings.
/admin PUT Update current configuration settings.

Authentication¶

Newton Inventory Service makes use of Rackspace’s internal Identity service for authentication and autthorization. Clients must use a Rackspace account that is a member of the nis_viewer or nis_admin LDAP groups for read only or read/write access, respectively.

Using an authorized account, retrieve a token from Identity by sending a POST request to the following url:

Production https://identity-internal.api.rackspacecloud.com/v2.0/tokens

Staging https://staging.identity-internal.api.rackspacecloud.com/v2.0/tokens

with a payload containing the authorized user credentials:

{
   "auth": {
      "RAX-AUTH:domain": { "name": "Rackspace" },
      "passwordCredentials": {
         "username": "<nis_user_account>",
         "password": "<password>"
      }
   }
}

The response should resemble the block below:

{
   "access": {
      "token": {
         "id": "aaaaa-bbbbb-ccccc-dddd",
         "expires": "2012-04-13T08:15:00.000-05:00"
      },
      "user": {
         "roles": [
            { "name": "Racker" },
            { "name": "Support" },
            { "name": "nis_admin" }
         ],
         "name": "jqsmith"
      }
   }
}

Once an auth token is obtained, include the value from access.token.id as a header named ‘X-Auth-Token’ in every call made to the NIS service.

Products¶

GET /products¶

Returns a list of all registered Products as an array.

Example request:

GET http://inventory.encore.rackspace.com/1.0/products HTTP/1.1
X-Auth-Token: aaaaa-bbbbb-ccccc-dddd
Accept: application/json

Example response:

HTTP/1.1 200 OK
Content-Type: application/json

{
   "data": [
      {
         "product_id": "123456",
         "product_name": "Perfomance One",
         "device_type": "Server"
      },
      {
         "product_id": "98765",
         "product_name": "5505 FW",
         "device_type": "Firewall"
      }
   ]
}
Query Parameters:
 
  • product_name – display name of the Product
  • product_id – string id of the Product
  • device_type – Product’s Device type (i.e. ‘server’ or ‘firewall’)
Status Codes:
  • 200 – OK
  • 400 – bad request
POST /products¶

Register a Product, returning the new Product if successful.

Example request:

POST http://inventory.encore.rackspace.com/1.0/products HTTP/1.1
X-Auth-Token: aaaaa-bbbbb-ccccc-dddd
Accept: application/json
Content-Type: application/json

{
   "product_id": "123456",
   "product_name": "Perfomance One",
   "device_type": "Server"
}

Example response:

HTTP/1.1 201 Product created
Content-Type: application/json

{
   "data": {
      "product_id": "123456",
      "product_name": "Perfomance One",
      "device_type": "Server"
   }
}
Request Headers:
 
  • Content-Type – header must be set to application/json
Json Parameters:
 
  • product_name (str) – display name of the Product; must be distict across all Products
  • product_id (str) – string id of the Product; must be distict across all Products
  • device_type (str) – Product’s Device type (i.e. ‘Server’ or ‘Firewall’);
Query Parameters:
 
  • none – This call takes no parameters
Status Codes:
  • 201 – Product created
  • 400 – bad request

PUT /products/(str: product_id)¶

Update the details of the Product having specified (str:product_id), returning the new Product if successful. May supply complete or partial Product object which will be merged on top of exisiting Product.

Example request:

PUT http://inventory.encore.rackspace.com/1.0/products/123456 HTTP/1.1
X-Auth-Token: aaaaa-bbbbb-ccccc-dddd
Accept: application/json
Content-Type: application/json

{ "device_type": "Toaster" }

Example response:

HTTP/1.1 201 Product updated
Content-Type: application/json

{
   "data": {
      "product_name": "Perfomance One",
      "product_id": "123456",
      "device_type": "Toaster"
   }
}
Request Headers:
 
  • Content-Type – header must be set to application/json
Json Parameters:
 
  • product_name (str) – display name of the Product; must be distict across all products
  • product_id (str) – string id of the Product; must be distict across all products
  • type (str) – Product’s Device type (i.e. ‘Server’ or ‘Firewall’);
Query Parameters:
 
  • none – This call takes no parameters
Status Codes:
  • 201 – Product updated
  • 400 – bad request
  • 404 – Product not found
DELETE /products/(str: product_id)¶

Ungregister the Product having specified (str:product_id) if no Devices belonging to this Product family currently exist.

Example request:

DELETE http://inventory.encore.rackspace.com/1.0/products/123456 HTTP/1.1

Example response:

HTTP/1.1 204 Product deleted
Parameters:
  • product_id (str) – product’s unique id
Query Parameters:
 
  • none – This call takes no parameters
Status Codes:
  • 204 – Product deleted
  • 400 – bad request
  • 403 – Devices using this Product still exist
  • 404 – Product not found

Deploy¶

POST /deploy¶

Request (int:quantity) number of Devices with Product type = (str:product_id) from datacenter = (str:dc) and optionally from aggr zone = (str:aggr_zone). If found, flag these devices as allocated and return an array of the device IDs. All devices returned will be allocated from the same aggr zone.

Example request:

POST http://inventory.encore.rackspace.com/1.0/deploy HTTP/1.1
X-Auth-Token: aaaaa-bbbbb-ccccc-dddd
Accept: application/json
Content-Type: application/json

{
   "dc" : "ORD1",
   "aggr_zone": "Aggr2.ORD1",
   "devices": [
      {
         "product_id": ""12345"",
         "quantity": 2
      },
      {
         "product_id": ""777777"",
         "quantity": 1
      }
   ]
}

Example response:

HTTP/1.1 200 OK
Content-Type: application/json

{
   "data": [
      {
         "device_id": 111111,
         "device_type": "server",
         "core_template_id": 102938
      },
      {
         "device_id": 22222,
         "device_type": "server",
         "core_template_id": 102938
      },
      {
         "device_id": 77777,
         "device_type": "firewall",
         "core_template_id": 987654
      },
   ]
}
Query Parameters:
 
  • none – This call takes no parameters
Status Codes:
  • 200 – no error
  • 404 – insufficient inventory

PUT /admin¶

Dynamically update service configuration. Include in body payload and option and its new value. Current options include:

aggr_round_robin:

Set device deployment distribution configuration.

true - device deployment will be balanced, always pulling from aggr with highest availablility that satisfies request.

false - device deployment will be consolidated, always pulling from aggr with lowest availablility that satisfies request.

Example request:

PUT http://inventory.encore.rackspace.com/1.0/admin HTTP/1.1
X-Auth-Token: aaaaa-bbbbb-ccccc-dddd
Accept: application/json
Content-Type: application/json

{
   "aggr_round_robin": true
}

Example response:

HTTP/1.1 200 OK
Content-Type: application/json

{
   "aggr_round_robin": true
}
Query Parameters:
 
  • none – This call takes no parameters
Json Parameters:
 
  • aggr_round_robin (str) – device deployment distribution
Status Codes:
  • 200 – no error
